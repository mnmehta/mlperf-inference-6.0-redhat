Usage
-----

To recreate this Grafana setup on a new OpenShift cluster:

1. Copy the entire mlperf_installer directory to the new cluster environment

2. Run the installation script:
   cd /path/to/mlperf_installer
   ./install_grafana.sh

3. Access Grafana at the route URL displayed after installation

4. Login with: root / start

5. Change the admin password immediately!

For detailed instructions, see GRAFANA_INSTALLATION.md

Notes
-----

- The service account token will be regenerated on the new cluster
- The Thanos querier URL will be auto-detected for the new cluster
- Dashboard UIDs will change but content will be preserved
- The route hostname will reflect the new cluster's domain

Dependencies
------------

The installation requires:
- OpenShift cluster (4.x)
- Cluster admin access (for ClusterRoleBinding)
- OpenShift monitoring stack enabled (default)
- oc CLI or kubectl configured

What Gets Installed
-------------------

1. Grafana Operator v5.21.2 (from community-operators)
2. Grafana instance v12.3.0
3. Service account with cluster-monitoring-view permissions
4. Prometheus datasource connected to OpenShift Thanos
5. 4 pre-configured dashboards for LLM-D monitoring
6. OpenShift route for external HTTPS access

Security Notes
--------------

- Default admin password (start) should be changed immediately
- Service account token is long-lived (10-year expiry)
- Route uses edge TLS termination
- Consider implementing OAuth/LDAP for production

Dashboard Descriptions
----------------------

1. Inference Gateway Dashboard
   - Monitors Gateway API Inference Extension metrics
   - Request rates, latencies, pool health, routing decisions
   - Source: https://github.com/kubernetes-sigs/gateway-api-inference-extension

2. LLM-D vLLM Overview
   - High-level vLLM deployment metrics
   - Token generation, throughput, GPU utilization

3. LLM-D Diagnostic Drilldown
   - Detailed per-pod diagnostics
   - Queue depths, cache hit rates, latency breakdowns

4. LLM-D Failure Saturation
   - Error rates by type
   - Resource and queue saturation indicators
   - System bottleneck analysis
